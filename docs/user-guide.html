insight Analytics Workbench

insight Analytics란 무엇인가

insight Analytics Workbench는 Tableau의 시각화 능력, JMP의 통계 분석 깊이, ChatGPT 스타일의 자연어 인터페이스를 하나의 브라우저 앱으로 통합한 로컬-퍼스트 데이터 분석 도구입니다.

Python, R, Excel 없이도 — 심지어 인터넷 없이도 — 데이터를 불러오고, 정제하고, 시각화하고, 통계 검정을 수행하고, 머신러닝 모델을 훈련할 수 있습니다.

Data 모드 — 데이터 탐색

데이터를 처음 받았을 때 가장 먼저 방문하는 모드입니다. 데이터 구조, 값의 분포, 품질 문제를 빠르게 파악합니다.

중앙 탭

탭내용
Data Preview전체 데이터를 고밀도 그리드로 표시. 정렬, 필터, 컬럼 고정, 글로벌 검색 가능.
Profiling컬럼별 요약 카드 — 데이터 타입, 결측률, 분포 미니 히스토그램, 주요 통계.

그리드 조작법

  • 컬럼 헤더 클릭 → 오름차순/내림차순 정렬
  • 컬럼 헤더 우클릭 또는 ▾ 메뉴 → 필터, 고정(Freeze), 숨기기(Hide) 옵션
  • 숫자 컬럼 필터: 범위(min~max) 입력
  • 범주형 컬럼 필터: 값 목록에서 다중 선택
  • 검색창: 모든 컬럼에서 텍스트 검색
  • 컬럼 아이콘 헤더: Dimension(범주형), Measure(수치형)

컬럼 상세 프로파일 (우측 패널)

좌측 탐색기 또는 그리드에서 컬럼명을 클릭하면 우측에 상세 프로파일이 표시됩니다.

  • 수치형 컬럼: 히스토그램 + 박스플롯 + 평균/중앙값/표준편차/사분위수
  • 범주형 컬럼: 상위 값 목록 + 빈도 막대
  • 날짜형 컬럼: 날짜 범위 + 월별 분포

Clean 모드 — 데이터 정제

실제 데이터에는 항상 오류가 있습니다. Clean 모드는 원본 데이터를 보존하면서 정제 작업을 단계별로 쌓아가는 비파괴적 파이프라인을 제공합니다.

이슈 바 (상단)

화면 상단의 이슈 바에서 문제를 한 눈에 파악하고 원클릭으로 수정할 수 있습니다.

이슈 유형감지 방법원클릭 해결
결측치 (Missing)null / 빈 값 셀 수평균값 채우기 / 해당 행 제거
중복 (Duplicates)완전히 동일한 행 수중복 행 제거
이상치 (Outliers)IQR × 1.5 벗어나는 값이상치 행 제거

수동 작업 추가 (우측 패널)

기본 정제

  • 컬럼 선택 → 적용할 작업 선택
  • 결측 채우기: 평균값(mean) / 중앙값(median) / 최빈값(mode) 중 선택
  • 이름 변경(Rename): 컬럼명 수정
  • 값 치환(Replace): 특정 값을 다른 값으로 일괄 교체
  • 타입 변환(Change Type): 문자열 ↔ 숫자 ↔ 날짜 변환
  • Drop column: 해당 컬럼을 데이터에서 완전 삭제 (ML 불필요 컬럼 제거에 유용)

인코딩 (Encoding)

머신러닝 모델은 숫자 입력만 받습니다. 문자열 컬럼을 숫자로 변환하는 두 가지 방법을 제공합니다.

방법동작적합한 경우
Label Encode 고유값을 알파벳 순으로 정렬 후 0, 1, 2… 정수 부여. 컬럼명_enc 새 컬럼 추가. 순서가 있는 범주 (예: 소형/중형/대형 → 0/1/2)
Dummy Encode 고유값마다 0 또는 1짜리 컬럼 생성 (One-Hot). 고유값 20개 초과 시 경고 팝업. 순서 없는 범주 (예: 구 이름, 건물 유형)

수치 변환 (Numeric Transform)

숫자형 컬럼에만 활성화됩니다.

변환수식사용 목적
Z-Score(x − μ) / σ스케일 통일 (회귀·군집 전처리)
Min-Max(x − min) / (max − min) → 0~1신경망, KNN 등 거리 기반 모델
Log(1+x)log₁₊(x)오른쪽 꼬리 왜도 완화 (가격, 거래량 등)
Rank오름차순 순위 1, 2, … n비모수 방법, 이상치 영향 제거
Bins (등폭 구간)N등분 → 컬럼_bin 범주 컬럼연속값 → 구간 범주화 (히스토그램 전처리)
Winsorize p%하위/상위 p% 분위수로 클리핑극단 이상치 영향 축소 (제거 없이)

Formula Column — 파생 컬럼 생성

JavaScript 수식으로 새 컬럼을 즉시 만들 수 있습니다. row 객체로 해당 행의 모든 컬럼 값에 접근합니다.

예시 수식설명
row.price_manwon * 1.05가격에 부가세 5% 추가
row.price_manwon / row.area_m2㎡당 가격 계산
Math.log1p(row.price_manwon)로그 변환 파생 컬럼
row.price_per_m2 > 1000 ? 'high' : 'low'조건 분기로 범주 컬럼 생성
row.district + '_' + row.building_type두 컬럼 결합

수식 오류가 발생한 셀은 null로 처리되며 파이프라인 전체에 영향을 주지 않습니다.

파이프라인 관리 (Undo/Redo)

  • 모든 작업은 우측 Pipeline 목록에 기록됩니다
  • 이전 단계 클릭 → 해당 시점으로 타임트래블 (이후 단계는 취소선으로 표시)
  • TopBar의 ↩ / ↪ 버튼으로 Undo/Redo
  • 원본 데이터는 절대 수정되지 않습니다
정제 결과는 모든 다른 모드(Chart, Stats, ML 등)에 자동 반영됩니다. ML 모드에서 학습 전 Label Encode와 Z-Score를 먼저 적용하면 정확도가 크게 향상됩니다.

SQL 모드 — 쿼리

복잡한 집계나 필터링이 필요할 때 SQL을 사용합니다. 결과를 새 데이터셋으로 저장해 다른 모드에서 활용할 수 있습니다.

지원 문법

SELECT district, AVG(price_per_m2) AS avg_ppm, COUNT(*) AS cnt
FROM seoul_txns
WHERE building_type = '아파트'
GROUP BY district
ORDER BY avg_ppm DESC
LIMIT 10

기본 SELECTSELECT col1, col2 FROM dataset

JOIN / 서브쿼리미지원 (예정)

실행 및 저장

  • ⌘ Enter (Mac) / Ctrl Enter (Windows) → 쿼리 실행
  • 결과가 하단 그리드에 표시됩니다
  • Save as Dataset 버튼 → 결과를 새 데이터셋으로 저장 (탐색기에 즉시 추가)

활용 예시

-- 구별 평균 가격 및 거래량 (상위 5구)
SELECT district,
       ROUND(AVG(price_manwon), 0) AS avg_price,
       ROUND(AVG(price_per_m2), 1) AS avg_ppm,
       COUNT(*) AS txn_count
FROM seoul_txns
GROUP BY district
ORDER BY avg_price DESC

Stats 모드 — 통계 분석

JMP 수준의 정밀 통계 분석을 제공합니다. 8개 탭으로 구성되며, 모든 결과에는 자동 해석 패널과 다음 분석 제안이 표시됩니다.

공통 결과 패널

Interpretation Panel (파란색)
분석 결과를 비전문가도 이해할 수 있는 언어로 자동 해석합니다. Insight Engine이 생성합니다.
Next Step Panel (초록색)
현재 분석 결과를 바탕으로 다음에 수행하면 좋을 분석을 제안합니다.

탭 1 — Descriptive (기술통계)

선택한 데이터셋의 모든 수치 컬럼에 대한 기술통계를 테이블로 표시합니다.

통계량설명
N / Missing유효한 값 수 / 결측값 수
Mean / Median평균값 / 중앙값
Std표본 표준편차
Min / Max최솟값 / 최댓값
Q1 / Q31사분위수 / 3사분위수
Skewness왜도 (0 = 대칭, 양수 = 오른쪽 꼬리, 음수 = 왼쪽 꼬리). |값| > 1.5이면 경고 색상
Kurtosis초과 첨도 (0 = 정규분포, 양수 = 뾰족, 음수 = 납작)

탭 2 — Distribution (분포 분석)

단일 컬럼의 분포를 심층 시각화합니다. JMP의 Distribution Platform에서 영감을 받았습니다.

1
컬럼 선택
우측 패널의 드롭다운에서 분석할 수치 컬럼 선택
2
히스토그램 확인
데이터를 구간(bin)으로 나눈 빈도 막대 차트. X축은 구간 레이블, Y축은 빈도수.
3
박스플롯 확인
IQR 기반 박스플롯 (가로 방향). 빨간 점 = IQR×1.5 범위 밖 이상치.
4
8개 통계 카드 읽기
n, Missing, Mean, Median, Std, IQR, Skewness, Kurtosis를 카드 형식으로 확인
왜도(Skewness) 해석 가이드
0에 가까울수록 대칭 분포. 양수(+): 오른쪽 꼬리 (고가 거래 소수 존재). 음수(-): 왼쪽 꼬리. |1.5| 초과 시 변환(log, sqrt) 검토 권장.

탭 3 — Correlation (상관 분석)

수치형 컬럼 쌍 간의 Pearson/Spearman 상관계수를 히트맵으로 표시합니다.

  • 색상: 빨강 = 강한 양의 상관(r→+1), 파랑 = 강한 음의 상관(r→-1)
  • 셀 클릭 → 해당 두 컬럼의 산점도 표시
  • |r| > 0.7: 강한 상관 / 0.4~0.7: 중간 / < 0.4: 약한 상관

탭 4 — T-Test

두 그룹 간 평균 차이가 유의미한지 검정합니다. (예: 아파트 vs 오피스텔 가격 차이)

  • 수치 컬럼 + 범주형 컬럼(2개 그룹) 선택
  • Welch t-test (등분산 가정 없음) 적용
  • p < 0.05: 유의미한 차이 Significant
  • Cohen's d: 효과 크기 (0.2=소, 0.5=중, 0.8=대)

탭 5 — ANOVA

3개 이상 그룹 간 평균 차이를 F-검정으로 비교합니다. (예: 구별 평균 가격 차이)

  • 수치 컬럼 + 범주형 컬럼(3개+ 그룹) 선택
  • F-통계량, p-값, η² (에타제곱, 효과 크기) 표시
  • 그룹별 평균 막대 차트로 시각화

탭 6 — Chi-Square

두 범주형 변수 사이의 독립성을 검정합니다. (예: 건물 유형과 거래 구역이 관련 있는가)

  • 범주형 컬럼 두 개 선택
  • 분할표(Contingency Table) + χ² 통계량 + Cramér's V (연관성 강도)

탭 7 — Regression (선형 회귀)

하나 이상의 변수로 수치 변수를 예측합니다. (예: 면적, 층수, 건축년도로 가격 예측)

  • Target (예측 대상) + Input (예측 변수들) 선택
  • R² (설명력), 수정 R², F-검정 표시
  • 계수 테이블: 각 변수의 계수, 표준오차, t-통계량, p-값, 유의성 표시
  • 예측 vs 실제 산점도

탭 8 — Analysis Builder (자동 분석)

JMP의 Analysis Builder에서 영감을 받은 탭입니다. 컬럼만 선택하면 적합한 분석을 자동으로 골라 실행합니다.

1
Target 컬럼 선택
분석하고 싶은 결과 변수(종속 변수)를 선택합니다. 수치형 또는 범주형 모두 가능합니다.
2
Input 컬럼 선택
원인/영향 변수(독립 변수)를 체크박스로 다중 선택합니다.
3
Run Analysis 클릭
컬럼 타입 조합에 따라 자동으로 분석 방법 선택:
  • 수치형 Target + 수치형 Input → OLS 선형 회귀
  • 수치형 Target + 범주형 Input → ANOVA
  • 범주형 Target + 범주형 Input → 카이제곱 검정
4
결과 확인
Summary / Visual / Statistical Results / Next Step 4개 섹션으로 구성된 결과 확인
Analysis Builder는 통계 방법을 모를 때 특히 유용합니다. "price_manwon를 예측하고 싶은데 어떤 방법을 써야 하지?"라는 상황에서 컬럼만 선택하면 됩니다.

ML 모드 — 머신러닝

Python이나 scikit-learn 없이 브라우저에서 직접 머신러닝 모델을 훈련하고 결과를 해석합니다.

Task 선택

Regression (회귀)
수치 값을 예측. 예: 아파트 가격 예측. OLS 정규방정식 기반.
Classification (분류)
범주를 예측. 예: 건물 유형 분류. k-NN 알고리즘, 표준화 자동 적용.
Clustering (군집화)
데이터를 자동으로 그룹화. KMeans 알고리즘. K값 직접 설정.

모델 훈련 절차

1
Task 선택
우측 패널 상단에서 Regression / Classification / Clustering 중 하나 선택
2
Target 컬럼 설정 (회귀/분류)
예측할 컬럼 선택. 회귀 = 수치형, 분류 = 범주형.
3
Feature 컬럼 선택
체크박스로 예측에 사용할 입력 변수들 선택. 수치형 컬럼만 사용 가능.
4
하이퍼파라미터 설정
분류: k (이웃 수, 기본값 5). 군집화: K (군집 수, 기본값 3). Train/Test 비율 선택.
5
Train 버튼 클릭
훈련 완료 후 결과 자동 표시. Model History에 자동 기록.

결과 해석

회귀 (Regression) 결과

  • R²: 설명력 (1에 가까울수록 좋음. 0.7+ 양호, 0.9+ 우수)
  • RMSE: 평균 오차 크기 (낮을수록 좋음)
  • Feature Importance: 각 변수의 상대적 중요도 막대 차트
  • 예측 vs 실제 산점도: 대각선에 가까울수록 예측 정확도 높음

분류 (Classification) 결과

  • Accuracy: 전체 정확도
  • Macro F1: 클래스 불균형을 고려한 F1 점수의 평균
  • 혼동 행렬 (Confusion Matrix): 실제 vs 예측 클래스 교차표. 대각선 = 정답
  • 클래스별 Precision / Recall / F1 테이블: 각 클래스의 성능 상세
    • Precision(정밀도): 예측이 맞는 비율
    • Recall(재현율): 실제 해당 클래스를 찾아낸 비율
    • F1: Precision과 Recall의 조화 평균

군집화 (Clustering) 결과

  • Inertia: 군집 내 분산 (낮을수록 군집이 촘촘함)
  • 군집 크기: 각 군집의 데이터 포인트 수
  • 군집 특성 표: 각 군집의 특성별 평균값. 군집 간 차이를 파악하는 핵심 표.
  • 군집 산점도: 색상으로 군집 구분

Model Comparison History

우측 하단의 Model History 테이블에서 이번 세션에 훈련한 모든 모델을 비교할 수 있습니다. Task, Target, 성능 지표(Score)를 한 눈에 비교합니다. 최근 10개를 유지합니다.

활용 예시 1 — 서울 아파트 시장 구조 파악

데이터 탐색 → 시각화

어느 구가 가장 비싸고, 어떤 유형이 프리미엄을 받는가?

목표: 구별/건물유형별 가격 차이를 시각화하고 통계적으로 검증한다.
1
Data 모드 → Profiling 탭
seoul_txns 선택. Profiling 탭에서 price_per_m2의 결측률과 분포를 확인합니다. 왜도가 높은지 체크.
2
Clean 모드 → 이슈 처리
이슈 바의 Missing 클릭 → price_per_m2 결측값을 중앙값(median)으로 채웁니다. Outliers 클릭 → 이상치 행 제거.
3
Chart 모드 → 구별 가격 비교
district를 Columns에, price_per_m2(AVG)를 Rows에 드래그. Show Me에서 H-Bar 선택. Sort Desc 활성화.
4
Chart 모드 → 건물 유형 추가
building_type을 Marks ▸ Color에 드롭. 쌓인 그룹 막대로 유형별 차이 확인.
5
Stats 모드 → ANOVA
ANOVA 탭에서 수치 컬럼 = price_per_m2, 범주 = district 선택. p-값이 0.05 미만이면 구 간 가격 차이가 통계적으로 유의미함.

활용 예시 2 — 아파트 가격 예측 모델

ML → 분석

면적, 층수, 건축연도로 ㎡당 가격을 얼마나 잘 예측할 수 있나?

목표: OLS 회귀로 가격 예측 모델을 만들고 주요 영향 변수를 파악한다.
1
Stats → Analysis Builder
Target = price_per_m2, Input = area_m2, floor, built_year 선택 → Run Analysis. 자동으로 OLS 회귀 실행. R²와 계수 테이블 확인.
2
Interpretation Panel 읽기
파란색 해석 패널에서 "R² = X → 모델이 변동성의 X%를 설명한다"는 자동 해석을 확인. 초록색 Next Step 패널에서 다음 분석 제안 확인.
3
ML → Regression
ML 모드에서 Regression 선택. 같은 Feature로 훈련. Feature Importance 차트에서 어떤 변수가 가장 중요한지 확인.
4
Ask Insight로 요약
Ask Insight 열기 → "Summarize last analysis" 클릭 → 마지막 ML 결과 요약 확인.

활용 예시 3 — 분포 분석으로 데이터 왜곡 발견

Stats → 데이터 품질

가격 분포가 정규분포를 따르는가? 통계 분석 전 사전 검토.

목표: Distribution 탭으로 가격의 왜도/첨도를 파악하고 이상치를 시각화한다.
1
Stats → Distribution 탭
우측 패널에서 price_per_m2 선택. 히스토그램이 오른쪽으로 치우친다면(long right tail) 양의 왜도가 높음.
2
왜도/첨도 카드 확인
Skewness > 1.5: 강한 비대칭 → 회귀 전 log 변환 고려. 박스플롯의 빨간 점 = IQR 이상치 시각 확인.
3
Clean → 이상치 제거
이상치가 많다면 Clean 모드로 이동. 이슈 바의 Outliers 클릭 → Remove Outliers. 이후 Distribution 탭에서 다시 확인.

팁 & 주의사항

성능 팁

  • 데이터 정제 후 Stats/ML 분석 수행을 권장합니다. 이상치가 포함된 채로 회귀를 돌리면 R²가 낮게 나올 수 있습니다.
  • Dashboard 위젯이 많을수록 Cross Filter 계산량이 늘어납니다. 위젯 수를 적절히 유지하세요.
  • ML 모드에서 Feature 수가 너무 많으면 k-NN 분류 성능이 저하될 수 있습니다 (차원의 저주). 4~8개 Feature 권장.

데이터 관련

  • 세션 새로고침 = 모든 상태 초기화. 클리닝 파이프라인, SQL 저장 데이터셋, 훈련된 모델이 모두 사라집니다.
  • 오프라인에서는 Map 모드가 버블맵 모드로 자동 전환됩니다 (GeoJSON CDN 미접근).
  • 샘플 데이터(seoul_txns)에는 의도적으로 결측치, 중복, 이상치가 포함되어 있습니다. Clean 모드 실습용입니다.

통계 해석 주의

  • p < 0.05 = "통계적으로 유의미한 차이가 있음"이지, "실용적으로 큰 차이"를 의미하지 않습니다. 효과 크기(Cohen's d, η²)도 함께 확인하세요.
  • Regression R²가 높더라도 인과관계를 의미하지 않습니다. 상관관계만 나타냅니다.
  • k-NN 분류는 이상치와 스케일 차이에 민감합니다. 이상치 제거 후 훈련을 권장합니다.
