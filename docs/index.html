프로젝트 개요

INSIGHT Analytics Workbench는 로컬-퍼스트 브라우저 전용 비즈니스 인텔리전스 도구입니다. Node.js, npm, 빌드 단계 없이 python3 -m http.server로 즉시 실행됩니다. 서울 아파트 실거래가 데이터를 포함한 샘플 데이터셋으로 즉시 사용 가능하며, 21종의 차트 타입과 8개의 분석 모드를 제공합니다.

목적 및 설계 철학

Zero Dependency
npm install 없음. CDN에서 React/ECharts를 로드하는 단일 HTML 파일 기반.
Local-First
데이터가 브라우저를 벗어나지 않음. 백엔드 서버 없이 모든 분석이 클라이언트에서 실행.
Tableau-Inspired UX
드래그&드롭 Shelf 기반 시각화. Columns/Rows/Color 드롭존으로 차트 즉시 구성.
In-Browser Babel
JSX가 브라우저에서 직접 트랜스파일됨. 개발자는 .jsx 파일만 편집하면 됨.

브랜드 아이덴티티

로고는 IN + SIGHT + Analytics 세 요소로 구성됩니다.

// 컴포넌트에서 상태 구독 (selector 패턴)
const mode = useStore(s => s.mode);
const viz  = useStore(s => s.viz);

// 액션으로 상태 변경
actions.setMode("visualize");
actions.setViz({ type: "bar" });
actions.addToShelf("cols", field);

// 파생 데이터 (항상 이 방법으로 데이터 접근)
const { ds, rows, columns } = Store.derive.getActiveData(datasetId);

데이터 엔진 — data.js

빌드 없이 즉시 사용 가능한 샘플 데이터셋을 생성하고 window.NODE로 노출합니다. 모든 데이터는 시드 기반 PRNG로 생성되므로 리로드 시에도 동일한 값을 반환합니다.

window.NODE 인터페이스

window.NODE = {
  datasets: Array<Dataset>,    // 4개 데이터셋 배열
  fmtWon: (v) => string,        // 만원 → "3.2억" / "1,500만" 형식
  fmtNum: (v, d?) => string,    // 숫자 → 로케일 포맷
  fmtCompact: (v) => string,   // 숫자 → "12.3억" / "4.5k" 압축 형식
  round: (n, d?) => number,     // 소수점 d자리 반올림
  palette: string[],            // 8개 CSS 변수명 ["--cat-1" .. "--cat-8"]
};

Dataset 스키마

type Dataset = {
  id:      string,           // 고유 식별자 (예: "seoul_txns")
  name:    string,           // 파일명 표시용 (예: "Seoul_Apartment_Txns.csv")
  short:   string,           // 축약명 (StatusBar 표시용)
  icon:    string,           // 아이콘 이름 ("table" | "trend" | "map")
  source:  string,           // "CSV" | "Derived" | "Simulated"
  rows:    object[],         // 실제 데이터 행 배열
  columns: Column[],         // 컬럼 메타데이터 배열
}

type Column = {
  key:   string,             // 행 객체의 키 이름
  label: string,             // 표시 이름
  type:  "integer" | "float" | "datetime" | "category" | "string",
  role:  "dimension" | "measure",
  agg:   string | null,      // 기본 집계함수 ("sum" | "avg" | "max" | "min")
  unit:  string | null,      // 단위 표시용 (예: "m²", "%")
  fmt:   string | null,      // 특수 포맷 ("won" → fmtWon 적용)
}

PRNG 함수 (mulberry32)

// 시드 기반 의사난수 생성기 — 리로드 시 동일한 데이터 보장
function rng(seed) {
  return function() {
    seed |= 0; seed = (seed + 0x6D2B79F5) | 0;
    let t = Math.imul(seed ^ (seed >>> 15), 1 | seed);
    // ... 고품질 32비트 난수 반환
    return ((t ^ (t >>> 14)) >>> 0) / 4294967296;
  };
}

전역 스토어 — store.jsx

Redux-like 단방향 데이터 흐름. window.Store로 노출되며, 모든 UI 상태와 데이터 변환 로직을 담당합니다.

window.Store 인터페이스

window.Store = {
  useStore: (selector?) => state,   // React hook: 상태 구독
  getState: () => state,             // 현재 스냅샷 (비리액티브)
  setState: (patch | fn) => void,   // 직접 상태 업데이트
  actions: { ... },                  // 모든 상태 변경 액션
  derive: { ... },                   // 파생 데이터 & 변환 함수
  stat: { ... },                     // 통계 함수 (순수함수)
  aggFn: { ... },                    // 집계 함수 맵
};

통계 수학 — statsMath.js

Stats 모드에서 사용하는 고정밀 수치 계산 함수들입니다. 외부 라이브러리 없이 순수 JS로 구현됩니다. window.SM으로 노출됩니다.

window.SM 인터페이스

함수설명
tP(t, df)Student t-분포 양측 p-값
fP(F, d1, d2)F-분포 상측 p-값
chiP(x, df)카이제곱 상측 p-값
matInverse(M)정방 행렬 역행렬 (가우스-조르당 소거법)
skewness(a)표본 왜도 (n/((n-1)(n-2)) 보정). n<3이면 null 반환.
kurtosis(a)Fisher 초과 첨도 (excess kurtosis, sample-corrected). n<4이면 null 반환.

내부적으로 Lanczos 근사 감마함수(gammln), 정규화 불완전 감마함수(gser/gcf), 불완전 베타함수(betai)를 사용합니다.

왜도 해석: |skewness| > 1.5이면 분포가 심하게 치우쳐 있음을 의미합니다. Descriptive 테이블에서 해당 값에 경고 색상이 표시됩니다. 첨도 해석: 0 = 정규분포 (초과 첨도 기준), 양수 = 뾰족한 분포(leptokurtic), 음수 = 납작한 분포(platykurtic).

테마 시스템

CSS 속성 선택자로 테마를 전환합니다. JS에서 document.documentElement.setAttribute("data-theme", "dark")만 호출하면 전체 색상이 전환됩니다.

/* 다크 테마 (기본) */
:root, [data-theme="dark"] { --bg-0: oklch(0.20 ...); ... }

/* 라이트 테마 */
[data-theme="light"] { --bg-0: oklch(0.955 ...); ... }

/* 색조 변형 (tone) */
[data-tone="warm"] { /* 배경에 따뜻한 색조 */ }

/* 밀도 (density) */
[data-density="comfortable"] { --row-h: 36px; }
ECharts의 테마는 별도 ECharts theme 시스템이 아닌 Charts.resolveVar()로 CSS 변수를 읽어 option에 직접 주입합니다. 따라서 테마 전환 시 차트가 자동으로 업데이트됩니다.

핵심 개발 규칙

  • window.*로 반드시 노출하기
    모든 컴포넌트와 유틸리티는 파일 말미에 window.X = X 또는 Object.assign(window, {...})로 끝나야 합니다. 이 줄이 없으면 다음 파일에서 참조할 수 없습니다.
  • 데이터는 항상 getActiveData()로 읽기
    NODE.datasets[i].rows 직접 접근 금지. 반드시 Store.derive.getActiveData(id)를 통해 읽어야 클리닝 스텝이 적용된 데이터가 반환됩니다.
  • animation: false 항상 유지
    모든 ECharts 옵션 최상위에 animation: false를 포함해야 합니다. iframe/Preview 환경에서 rAF 루프가 멈추면 차트가 흰 캔버스로 보이는 버그가 발생합니다.
  • 하드코딩 색상 절대 금지
    #e8611a, rgb(100,150,200) 같은 색상 리터럴 사용 금지. 반드시 var(--accent), Charts.resolveVar("--cat-1")을 통해 사용하세요.
  • <script> 순서 변경 금지
    index.html의 스크립트 로드 순서는 의존성 순서입니다. 순서를 바꾸면 window.X is not defined 에러가 발생합니다.
  • .jsx 파일 수정 후 저장 = 즉시 반영
    in-browser Babel이 매 페이지 로드 시 .jsx를 트랜스파일합니다. 별도 빌드 명령이 필요 없습니다. 브라우저 새로고침으로 변경사항을 확인하세요.
  • ECharts category axis에서 분수 인덱스 사용 금지
    api.coord([1.5, y])처럼 분수 카테고리 인덱스는 ECharts가 무시합니다. 폭 계산이 필요할 때는 픽셀 공간에서 직접 계산하세요 (Violin 차트 구현 참조).
  • 크로스파일 통신은 window.*와 CustomEvent만 사용
    ES module import/export 불가. 파일 간 데이터 전달은 window 전역 변수, 또는 window.dispatchEvent(new CustomEvent(...))로 처리합니다.
